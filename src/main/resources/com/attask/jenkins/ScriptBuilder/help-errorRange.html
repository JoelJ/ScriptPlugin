(All whitespace is ignored.)

When Less Than, Greater Than, or Exactly are set, this should be a single integer value.

When Non-Zero this value is completely ignored.

When Custom is set, then you can specify ranges and multiple values.
A range looks like: 10 > 20 This includes both the upper and lower bounds. So in this case, 10, 20, or any value in between would trigger the build to fail or become unstable.
Multiple values can be separated with a ",". For example: -10,20 will fail or become unstable if the exit code is either -10 or 20 and will pass with any other value.
Complex values can be used. For example: -10,10, 20 > 400, 500 > 1000
